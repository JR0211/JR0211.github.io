SEMANA 11

DEFINICIONES

- UseState:
Es un hook que te permite añadir estado a tus componentes funcionales. Antes de la introducción de los hooks en React (versiones anteriores a la 16.8), los componentes funcionales no tenían la capacidad de manejar estados locales.
- JavaScript:
Es un lenguaje de programación de alto nivel, interpretado y orientado a objetos, diseñado principalmente para ejecutarse en navegadores web. Se utiliza para mejorar la interactividad de las páginas web al permitir la manipulación dinámica del contenido, responder a eventos del usuario y realizar operaciones en el lado del cliente. JavaScript es esencial en el desarrollo web moderno y se emplea para crear experiencias interactivas, validar formularios, gestionar el DOM y comunicarse con servidores a través de tecnologías como AJAX.
- API:
Conjunto de reglas y definiciones que permiten que diferentes aplicaciones se comuniquen entre sí. Una API define cómo los componentes de software deben interactuar, qué operaciones pueden realizar y qué datos pueden intercambiar. Las APIs pueden ser utilizadas para permitir la comunicación entre aplicaciones, servicios web, bibliotecas, o cualquier otro tipo de software.
- Axios:
Es una interfaz de procesamiento de aplicaciones entre un servidor web y un navegador web, para el ejemplo se uso una API en especifico.
- UseEffect:
Es otro hook fundamental que permite realizar efectos secundarios en componentes funcionales. Los efectos secundarios pueden ser tareas asíncronas, subscripciones a eventos, modificaciones del DOM y más. useEffect se utiliza para manejar estas operaciones en componentes funcionales.
- UseRef:
Se utiliza para crear una referencia mutable que puede asociarse con un elemento del DOM o con un valor mutable en el ciclo de vida del componente. A diferencia del estado, las referencias creadas con useRef no provocan una re-renderización cuando cambian.
- CustomHooks:
En React son funciones JavaScript que encapsulan lógica de estado y comportamiento para ser reutilizada en múltiples componentes. Estos hooks personalizados siguen la convención de nombre use para que React los reconozca como ganchos y puedan aprovechar las características de los hooks, como el estado local y los ciclos de vida.

PROCEDIMIENTOS
Durante esta semana, se exploraron aspectos fundamentales en React, centrándonos especialmente en el uso efectivo del hook useEffect para gestionar efectos secundarios en nuestras aplicaciones. Se profundizó en la modularización de la lógica de formularios mediante la creación y utilización de custom hooks, como por ejemplo, el hook useForm. Además, se adquirió la capacidad de interactuar con APIs externas mediante el uso de Axios, permitiéndonos realizar solicitudes HTTP de manera eficiente. Una parte importante del aprendizaje se centró en el uso de referencias (useRef), lo cual posibilitó el acceso directo a nodos específicos en el DOM. En la práctica, se implementaron funcionalidades como contadores y generadores de números aleatorios que interactuaban con APIs externas, demostrando la versatilidad de React en la gestión de datos dinámicos. Además, se enfatizó la importancia de la estructuración de componentes en archivos separados, lo que contribuye significativamente a la reutilización y el mantenimiento eficiente del código. Esta práctica facilita la comprensión y manipulación de componentes individuales, promoviendo una arquitectura más clara y sostenible en el desarrollo de aplicaciones React.
