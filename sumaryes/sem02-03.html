SEMANA 2 Y 3

DEFINICIONES
- HTML:
Significa "HyperText Markup Language", es un estándar que se utiliza para la creación y el diseño de páginas web. Fue desarrollado originalmente por el World Wide Web Consortium (W3C) con el objetivo de facilitar la creación y el intercambio de documentos en la World Wide Web.
- Arbol DOM:
Es una estructura de datos jerárquica y en forma de árbol que representa la estructura de un documento HTML o XML. El DOM se utiliza principalmente en el desarrollo web para permitir la manipulación dinámica y la interacción con los elementos de una página web mediante lenguajes de programación como JavaScript.
- Ciclo de vida de una página web:
Se refiere a las distintas etapas que experimenta una página desde que se carga inicialmente en un navegador web hasta que se cierra o se navega a otra página. Sus principales fases son: solicitud y descarga, procesamiento HTML y procesador DOM, construcción árbol CSSOM, renderizado, ejecución de scrpits, interactividad y cierre.
- CSS:
Significa "Cascading Style Sheets", es un lenguaje utilizado para describir el aspecto y formato de un documento HTML. En resumen, CSS se encarga de definir cómo se presentará el contenido HTML en un navegador.
- Layout:
Se refiere a la disposición o diseño estructurado de elementos en una página, documento o entorno gráfico. En el contexto del desarrollo web y diseño de interfaces, el "layout" describe la organización y la distribución de componentes, como texto, imágenes, formularios y otros elementos, en una página web.
- Responsivo:
También conocido como diseño web adaptable, se refiere a la capacidad de una página web para adaptarse y ofrecer una experiencia óptima independientemente del dispositivo o tamaño de pantalla en el que se visualice. El diseño responsivo utiliza técnicas de diseño y desarrollo que permiten que la estructura, el diseño y los elementos de una página se ajusten de manera dinámica para satisfacer las características específicas del dispositivo del usuario.
- Accesibilidad:
Es la práctica de diseñar productos, entornos y servicios de manera que sean utilizables por todas las personas, independientemente de sus habilidades físicas o cognitivas, y sin la necesidad de adaptaciones especiales.

PROCEDIMIENTOS
Toda pagina web sigue la estructura DOM que significa Document Object Model donde se detallan tres partes diferenciadas: cabecera, cuerpo y pie de página. Sus terminilogías o etiquetas dentro de HTML son: header, body y footer.
- Cabecera: es la parte superior de la página web. Aquí suele incluirse la información básica de la empresa o marca y es consistente en todo el sitio, es decir, se repite en cada página de la web por la que navegas.
- Cuerpo: es la parte de la web que alberga el contenido principal de tu página. Esta parte sí es diferente en cada página de tu sitio. Es decir, no será el mismo contenido el que incluyes en tu página de contacto que en la página principal.
- Pie de página: es la parte inferior de una página web. Al igual que la cabecera, se repite y es consistente en cada página. En el footer podemos encontrar menú simplificado, información de contacto, botones de redes sociales, política de privacidad.

El funcionamiento de HTML lo podemos evidenciar en la plataforma VS Code donde se estructuran el conjunto de etiquetas para que la página pueda funcionar con las características y contenido brindado.

Existe una amplia variedad de etiquetas que se utilizan en HTML, a continuación, se muestran algunas de las etiquetas más comunes:

.::: ARBOL DOM :::.
Es una representación jerárquica en forma de árbol de la estructura de un documento HTML o XML. Cada elemento, atributo y texto en el documento es representado como un nodo en el árbol, permitiendo la manipulación y la interacción dinámica con el contenido de la página mediante JavaScript u otras tecnologías del lado del cliente.

.::: HOJAS DE ESTILO :::.
Se refieren a las reglas y definiciones que se aplican para controlar el diseño y la presentación visual de un documento HTML o XML. Las hojas de estilo suelen estar escritas en CSS (Cascading Style Sheets), y determinan aspectos como el color, la tipografía, el espaciado, la disposición y otros estilos visuales de los elementos en una página web.

.::: FLEXBOX :::.
O Flexible Box Layout, es un modelo de diseño en CSS que permite diseñar de manera eficiente interfaces flexibles y dinámicas, especialmente para el diseño de cajas y la distribución de espacio entre elementos en un contenedor.

.::: GRID LAYOUT :::.
Es un modelo de diseño en CSS que permite crear interfaces bidimensionales, organizando el contenido en filas y columnas en una cuadrícula. Este sistema de diseño proporciona una forma más poderosa y precisa de estructurar y alinear elementos en comparación con Flexbox.
