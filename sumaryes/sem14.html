SEMANA 14

DEFINICIONES

- NextAuth:
Librería utilizada con Next.js para la implementación de autenticación y autorización en aplicaciones web.
- Backend:
Parte de una aplicación que se encarga de procesar datos y realizar operaciones del lado del servidor.
- Frontend:
Interfaz de usuario y experiencia de usuario de una aplicación, visible y accesible para el usuario final.
- Boostrap:
Marco de diseño que facilita el desarrollo de interfaces web responsivas y atractivas.
- Midleware:
Software que se encuentra entre el sistema operativo y las aplicaciones, facilitando la comunicación y el procesamiento de datos.
- Proveedor de sesiones:
Componente que gestiona la información de la sesión del usuario, como su estado de autenticación.

PROCEDIMIENTOS
Se abordó la implementación de autenticación con Next.js utilizando NextAuth. Se instaló y configuró el backend del sistema, se exploró la documentación de NextAuth para proveedores de credenciales y se procedió a crear el frontend con Bootstrap. La configuración de NextAuth incluyó la creación de proveedores de credenciales para autenticación mediante correo electrónico y contraseña. Se instaló y utilizó el paquete Bootstrap para mejorar el diseño y la apariencia del frontend. También se implementó la gestión de sesiones y se exploraron conceptos como el proveedor de sesiones y el middleware de NextAuth.

.::: Instalación y configuración :::.
Instalación y configuración del backend del sistema, incluyendo la ejecución de comandos como npm i y docker compose up -d.
Creación de un proveedor de sesiones para gestionar la autenticación y autorización del usuario.
Configuración de proveedores de credenciales para autenticación mediante correo electrónico y contraseña.
Instalación del paquete Bootstrap para mejorar el diseño y la apariencia del frontend.
Implementación de un componente de botones para manejar la autenticación y desautenticación del usuario.
Configuración de middleware para gestionar rutas protegidas en el dashboard de la aplicación.
RESULTADOS
Se logró instalar y configurar el backend del sistema, así como la creación del frontend utilizando Bootstrap para mejorar la interfaz de usuario.
NextAuth se implementó exitosamente para gestionar la autenticación mediante correo electrónico y contraseña.
Se diseñó y creó una página de dashboard protegida, la cual solo es accesible para usuarios autenticados.
Se implementaron botones de autenticación y desautenticación, permitiendo al usuario interactuar con el sistema de autenticación.
Se exploraron y aplicaron conceptos importantes, como el proveedor de sesiones y el middleware, para gestionar el estado de autenticación y proteger rutas específicas en la aplicación.
