DEFINICIONES
- HTML:
Significa "HyperText Markup Language", es un estándar que se utiliza para la creación y el diseño de páginas web. Fue desarrollado originalmente por el World Wide Web Consortium (W3C) con el objetivo de facilitar la creación y el intercambio de documentos en la World Wide Web.
- CSS:
Significa "Cascading Style Sheets", es un lenguaje utilizado para describir el aspecto y formato de un documento HTML. En resumen, CSS se encarga de definir cómo se presentará el contenido HTML en un navegador.
- HTTP:
Significa "Hypertext Transfer Protocol" (Protocolo de Transferencia de Hipertexto), es un protocolo de comunicación que se utiliza para la transmisión de información en la World Wide Web.
- JavaScript:
Es un lenguaje de programación de alto nivel, interpretado y orientado a objetos, diseñado principalmente para ejecutarse en navegadores web. Se utiliza para mejorar la interactividad de las páginas web al permitir la manipulación dinámica del contenido, responder a eventos del usuario y realizar operaciones en el lado del cliente. JavaScript es esencial en el desarrollo web moderno y se emplea para crear experiencias interactivas, validar formularios, gestionar el DOM y comunicarse con servidores a través de tecnologías como AJAX.
- Desarrollador Frontend:
Es un profesional de la programación que se enfoca en la creación y mantenimiento de la interfaz de usuario y la experiencia del usuario en aplicaciones web o sitios web. Este tipo de desarrollador trabaja en el lado del cliente, es decir, en la parte del software que interactúa directamente con los usuarios finales a través de sus navegadores web.
- Desarrollador Backend:
Es un profesional de la programación que se especializa en la creación y mantenimiento de la lógica, la base de datos y el servidor de una aplicación. A diferencia de los desarrolladores Frontend, que se centran en la interfaz de usuario y la interacción directa con los usuarios, los desarrolladores Backend trabajan en la "parte trasera" o el lado del servidor de una aplicación.
- Control de versiones:
Es un sistema que registra los cambios realizados en un conjunto de archivos a lo largo del tiempo. Su propósito principal es permitir la colaboración eficiente en proyectos de software y rastrear la evolución de los códigos fuente y otros archivos.
- EMMET:
Es una herramienta y conjunto de abreviaturas que se utiliza para acelerar y simplificar la escritura de código HTML y CSS en entornos de desarrollo web. Proporciona una sintaxis abreviada que se expande automáticamente en código HTML o CSS más completo.

PROCEDIMIENTOS
La sesión trató del uso de las abreviaturas de EMMETT para la agilización de codificación. Es una de las funciones integradas de VS Code que también funciona en otros editores como una extensión. Emmett es como una taquigrafía, se puede escribir con facilidad gran cantidad de código de trabajo en HTML y CSS.

Intente escribir desde un div o h1 hasta abreviaciones como bq para blockquote, btn para button, sect para section, etc.
.::: CLASES :::.
Para definir una clase junto con la etiqueta solo se realiza de la siguiente manera: Etiqueta.clase

.::: ID :::.
Para definir un Id se realiza de manera similar: Etiqueta#Id

.::: CONTENIDO :::.
Para incluir contenido dentro de la etiqueta, simplemente envolvemos el contenido entre llaves, { }.

.::: ETIQUETAS HERMANOS E HIJOS :::.
También podemos especificar etiquetas hermanos e hijos usando los caracteres + y > respectivamente.

.::: ESCALANDO :::.
En este ejemplo, "treparemos" por la estructura HTML usando ^.

.::: MULTIPLICACIÓN y $ :::.
Podemos generar múltiples etiquetas multiplicando (*) y numerando elementos en secuencia usando un signo de dólar ($).

Es un texto ficticio utilizado por los desarrolladores para representar datos en una página. Simplemente escriba lorem y presione Tab.

SEMANA 2 Y 3

28 de agosto al 1 de setiembre Y del 4 de setiembre al 8 de setiembre del 2023

DEFINICIONES
- HTML:
Significa "HyperText Markup Language", es un estándar que se utiliza para la creación y el diseño de páginas web. Fue desarrollado originalmente por el World Wide Web Consortium (W3C) con el objetivo de facilitar la creación y el intercambio de documentos en la World Wide Web.
- Arbol DOM:
Es una estructura de datos jerárquica y en forma de árbol que representa la estructura de un documento HTML o XML. El DOM se utiliza principalmente en el desarrollo web para permitir la manipulación dinámica y la interacción con los elementos de una página web mediante lenguajes de programación como JavaScript.
- Ciclo de vida de una página web:
Se refiere a las distintas etapas que experimenta una página desde que se carga inicialmente en un navegador web hasta que se cierra o se navega a otra página. Sus principales fases son: solicitud y descarga, procesamiento HTML y procesador DOM, construcción árbol CSSOM, renderizado, ejecución de scrpits, interactividad y cierre.
- CSS:
Significa "Cascading Style Sheets", es un lenguaje utilizado para describir el aspecto y formato de un documento HTML. En resumen, CSS se encarga de definir cómo se presentará el contenido HTML en un navegador.
- Layout:
Se refiere a la disposición o diseño estructurado de elementos en una página, documento o entorno gráfico. En el contexto del desarrollo web y diseño de interfaces, el "layout" describe la organización y la distribución de componentes, como texto, imágenes, formularios y otros elementos, en una página web.
- Responsivo:
También conocido como diseño web adaptable, se refiere a la capacidad de una página web para adaptarse y ofrecer una experiencia óptima independientemente del dispositivo o tamaño de pantalla en el que se visualice. El diseño responsivo utiliza técnicas de diseño y desarrollo que permiten que la estructura, el diseño y los elementos de una página se ajusten de manera dinámica para satisfacer las características específicas del dispositivo del usuario.
- Accesibilidad:
Es la práctica de diseñar productos, entornos y servicios de manera que sean utilizables por todas las personas, independientemente de sus habilidades físicas o cognitivas, y sin la necesidad de adaptaciones especiales.

PROCEDIMIENTOS
Toda pagina web sigue la estructura DOM que significa Document Object Model donde se detallan tres partes diferenciadas: cabecera, cuerpo y pie de página. Sus terminilogías o etiquetas dentro de HTML son: header, body y footer.
- Cabecera: es la parte superior de la página web. Aquí suele incluirse la información básica de la empresa o marca y es consistente en todo el sitio, es decir, se repite en cada página de la web por la que navegas.
- Cuerpo: es la parte de la web que alberga el contenido principal de tu página. Esta parte sí es diferente en cada página de tu sitio. Es decir, no será el mismo contenido el que incluyes en tu página de contacto que en la página principal.
- Pie de página: es la parte inferior de una página web. Al igual que la cabecera, se repite y es consistente en cada página. En el footer podemos encontrar menú simplificado, información de contacto, botones de redes sociales, política de privacidad.

El funcionamiento de HTML lo podemos evidenciar en la plataforma VS Code donde se estructuran el conjunto de etiquetas para que la página pueda funcionar con las características y contenido brindado.

Existe una amplia variedad de etiquetas que se utilizan en HTML, a continuación, se muestran algunas de las etiquetas más comunes:

.::: ARBOL DOM :::.
Es una representación jerárquica en forma de árbol de la estructura de un documento HTML o XML. Cada elemento, atributo y texto en el documento es representado como un nodo en el árbol, permitiendo la manipulación y la interacción dinámica con el contenido de la página mediante JavaScript u otras tecnologías del lado del cliente.

.::: HOJAS DE ESTILO :::.
Se refieren a las reglas y definiciones que se aplican para controlar el diseño y la presentación visual de un documento HTML o XML. Las hojas de estilo suelen estar escritas en CSS (Cascading Style Sheets), y determinan aspectos como el color, la tipografía, el espaciado, la disposición y otros estilos visuales de los elementos en una página web.

.::: FLEXBOX :::.
O Flexible Box Layout, es un modelo de diseño en CSS que permite diseñar de manera eficiente interfaces flexibles y dinámicas, especialmente para el diseño de cajas y la distribución de espacio entre elementos en un contenedor.

.::: GRID LAYOUT :::.
Es un modelo de diseño en CSS que permite crear interfaces bidimensionales, organizando el contenido en filas y columnas en una cuadrícula. Este sistema de diseño proporciona una forma más poderosa y precisa de estructurar y alinear elementos en comparación con Flexbox.

DEFINICIONES
- ECMAScript:
Es la especificación estándar que define el lenguaje de programación subyacente de JavaScript. Fue creado por Ecma International, una organización internacional de normalización, y se estableció por primera vez en 1997. La especificación ECMAScript proporciona las reglas y las pautas que los implementadores de JavaScript deben seguir para garantizar la compatibilidad entre diferentes motores de JavaScript en navegadores y entornos de ejecución.
- Desarrollador Frontend:
Es un profesional de la programación que se enfoca en la creación y mantenimiento de la interfaz de usuario y la experiencia del usuario en aplicaciones web o sitios web. Este tipo de desarrollador trabaja en el lado del cliente, es decir, en la parte del software que interactúa directamente con los usuarios finales a través de sus navegadores web.
- JavaScript:
Es un lenguaje de programación de alto nivel, interpretado y orientado a objetos, diseñado principalmente para ejecutarse en navegadores web. Se utiliza para mejorar la interactividad de las páginas web al permitir la manipulación dinámica del contenido, responder a eventos del usuario y realizar operaciones en el lado del cliente. JavaScript es esencial en el desarrollo web moderno y se emplea para crear experiencias interactivas, validar formularios, gestionar el DOM y comunicarse con servidores a través de tecnologías como AJAX.
- SpiderMonkey:
Es el motor de JavaScript desarrollado por Mozilla, la organización detrás del navegador web Firefox. Es un componente clave en el ecosistema de Mozilla y se utiliza para interpretar y ejecutar el código JavaScript en el navegador Firefox.
- NodeJS:
Es un entorno de ejecución de JavaScript del lado del servidor que utiliza el motor V8 de Google Chrome para ejecutar código. A diferencia de JavaScript en el navegador, que se ejecuta del lado del cliente, Node.js permite ejecutar JavaScript del lado del servidor, lo que posibilita el desarrollo de aplicaciones web escalables y de alto rendimiento.
- POO:
La Programación Orientada a Objetos (POO) es un paradigma de programación que se basa en el concepto de "objetos". En este enfoque, los objetos son instancias de clases que encapsulan datos y comportamientos relacionados. La POO se centra en organizar el código de manera que los datos y las funciones relacionadas estén agrupados en unidades cohesivas y reutilizables.
